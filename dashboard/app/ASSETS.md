# Asset attribution manifest

Every visual asset in the tycoon office-scene dashboard is **self-made** — authored
inline as code (SVG/CSS) for this app — so there is nothing here with third-party
licensing to track. No image files, no external fonts, no icon packs, no font
files are embedded anywhere in `dashboard/app/`.

## Why no external/CC0-sourced files

The task allowed either self-made assets or named CC0 sources. Self-making every
asset as inline SVG/CSS was the simpler, unambiguous choice: it needs zero binary
files in the repo, zero attribution bookkeeping, and zero risk of an unlicensed or
mis-licensed third-party asset slipping in. If a richer pixel-art style is wanted
later, replace `CAT_GRID` in `src/entities/cat/ui.tsx` with a taller/wider grid or
swap in real CC0 sprite sheets — update this manifest with the source + license
the moment any such file is added.
