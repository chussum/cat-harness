# Design Pixel QA — Figma 겹침 대조 루프

Figma 디자인과 구현 화면을 픽셀 단위로 맞추는 검증·수정 루프. 실제 QA 세션에서 수십 번의 시행착오로 확립된 절차다. **아래 철칙을 어기면 같은 실수를 반복하게 된다.**

## 철칙 (전부 실제 사고에서 나온 규칙)

1. **반드시 겹쳐서 육안으로 판정한다.** 수치 측정은 원인 추적용이고, 최종 판정은 항상 "두 이미지를 포갠 화면"을 눈으로 본다. 수치가 맞는데 눈에 다르면 측정이 틀린 것이다.
2. **두 캡쳐는 반드시 같은 픽셀 크기여야 한다.** Figma export가 375px면 구현 캡쳐도 375px. 뷰포트가 넓으면 `width:100%` 컴포넌트가 늘어나 캡쳐가 900px가 되고, 겹침이 전부 어긋나 보인다(실제 사고). 캡쳐 후 `sips`로 크기부터 대조하라.
3. **목데이터(사진·상품명·썸네일)는 회색으로 마스킹하고 비교한다.** 콘텐츠가 다르면 픽셀 diff가 전부 "다름"으로 나와 레이아웃 차이를 가린다. 겹침의 목적은 **글자·버튼의 위치/크기/간격**이다.
4. **가로만 재지 말고 세로도 잰다.** 세로 4px 오프셋을 가로 측정만 반복하며 못 찾는 사고가 있었다. 모든 요소를 가로 잉크 span + 세로 잉크 rows 둘 다 측정한다.
5. **비교하는 두 요소는 같은 색 판정식(threshold)으로 잰다.** 할인율(핑크/빨강)과 금액(검정)을 다른 판정식으로 재면 ±1px 가짜 오프셋이 나온다. 이걸 "보정"하면 멀쩡한 밑선을 망가뜨린다(실제 사고).
6. **크롬은 글자를 정수 픽셀에 스냅한다(DPR 1).** `position/top/margin` 0.5px 이동은 렌더에 반영되지 않는다(실측 확인). 위치 보정은 1px 단위, **폭 보정만 letter-spacing 소수점(0.1~0.15px)이 유효**하다.
7. **개별 텍스트에 세로 오프셋을 넣지 마라.** 같은 행의 두 텍스트(예: 할인율+금액)는 브라우저가 밑선을 자동 정렬한다. 한쪽에만 `top: 1px`을 넣으면 구현 화면 자체가 어긋난다. 행 전체(부모 row)를 함께 이동시켜라.
8. **±1 경계 안티앨리어싱 픽셀은 쫓지 마라.** 같은 CSS에서도 측정마다 301↔302로 흔들리는 픽셀이 있다. 이걸 맞추려 값을 바꾸면 다른 곳이 깨진다. 흔들림 이력을 기록하고 육안 겹침으로 판정한다.
9. **수정 1회 = 재캡쳐 + 재측정 + 겹침 1회.** 수정 후 검증 없이 다음 수정으로 넘어가지 않는다. 일치할 때까지 루프.
10. **Figma 수치(metadata)보다 잉크 실측이 우선한다.** Figma 노드 gap이 4px라 해도 실제 export 픽셀이 다르면 픽셀을 따른다. 단, 색상은 디자이너 확인 없이 바꾸지 마라(Figma가 옛 색일 수 있음 — 실제 사례: 할인율 핑크는 구버전, 현행 빨강 유지).

## 준비물

- **구현 렌더**: 검증 대상 컴포넌트를 격리해 띄운 화면 — 컴포넌트 뷰어, 전용 라우트, 또는 최소 재현 페이지 등 프로젝트에 맞는 무엇이든. 전체 앱 화면보다 **컴포넌트 격리 렌더가 픽셀 대조에 정확하다.**
- **Figma**: Figma MCP (`get_screenshot`으로 TO-BE export, `get_metadata`로 노드 구조/치수, `get_design_context`로 컴포넌트 매핑 — Code Connect로 디자인 시스템(DS, 공용 컴포넌트 모음) 컴포넌트가 지정된 경우 그 스펙이 정답).
- **캡쳐/측정**: Playwright MCP + 스크래치패드에 `python3 -m http.server 7020` (측정용 canvas HTML은 file:// 차단이라 HTTP로 서빙).

## 워크플로우

### 1. 캡쳐 (둘 다, 같은 크기)

```
# Figma TO-BE: get_screenshot(fileKey, nodeId) → curl로 저장
# 구현 AS-IS: 뷰포트를 Figma 프레임 폭(보통 375)으로 resize 후 element 캡쳐
browser_resize(375, 700)
browser_navigate("<컴포넌트를 격리 렌더한 URL>")   # 컴포넌트 뷰어 iframe / 전용 라우트 / 최소 재현 페이지
browser_evaluate: 캡쳐 대상 밖의 여백·배경 제거 CSS 주입 (컨테이너 margin/padding 0, 배경 흰색)
  'html,body{margin:0!important;padding:0!important;background:#fff!important;}
   <대상 컨테이너 셀렉터>{padding:0!important;margin:0!important;}'
browser_take_screenshot(target="<대상 컴포넌트 루트 셀렉터>", scale="device")
# 저장 후 반드시: sips로 두 PNG 크기 동일 확인
```

주의: Playwright 스크린샷은 **repo 루트**에 떨어진다 → 스크래치패드로 옮기고 `.playwright-mcp/` 삭제(깃 오염 방지).

### 2. 측정 (잉크 스캔 — references/measure-template.html)

canvas로 두 PNG를 읽어 **잉크 픽셀**(배경이 아닌 픽셀)의 가로 span/세로 rows를 요소별로 스캔한다. 색 판정식:

```js
const dark = (R,G,B)=>R<110&&G<110&&B<110;          // 검정 텍스트
const warm = (R,G,B)=>R>180&&G<130;                  // 빨강/핑크 (B 조건 넣으면 핑크 누락 — 실제 사고)
const blue = (R,G,B)=>B>150&&R<150&&B-R>40;          // 파랑 아이콘/텍스트
const white= (R,G,B)=>R>235&&G>235&&B>235;           // 흰 pill 배경
```

요소마다 `{ours: [L,R]/[top,bottom], figma: [...]}"` 표를 만들어 오프셋을 확정한다.

### 3. 원인 매핑 (자주 나오는 실제 원인)

| 증상 | 실제 원인 | 수정 |
|---|---|---|
| 아래 요소들이 전부 N px 위로 밀림 | DS Text 컴포넌트 기본 line-height < Figma 텍스트박스 높이 | 해당 텍스트에 `line-height: <Figma값>px` |
| 행 안의 그룹이 1~2px 밀림 | flex `align-items:center`의 높이 재분배 | `flex-start` + 명시적 gap으로 전환 |
| 텍스트 폭이 1px 좁음/넓음 | 브라우저 vs Figma 글리프 advance 차이 | `letter-spacing: ±0.1~0.15px` |
| 버튼이 큼/작음 | 커스텀 pill이 DS 표준 버튼 크기와 다름 | `box-sizing:border-box`+`height` 고정+패딩 계산 |
| 버튼 내용 쏠림 | Figma는 좌측 편향인데 구현은 가운데(또는 반대) | 비대칭 패딩으로 잉크 x좌표 맞춤 |
| 이미지가 텍스트 위/아래 반대 | DS BottomSheet(바텀시트) 컴포넌트의 `imagePosition` 류 속성 | `'top'`↔`'bottom'` |

### 4. 수정 → 재검증 루프

수정 → 재캡쳐 → 재측정 → **겹침 육안** → 다르면 반복. 겹침 확인은 두 가지 뷰:

- **색 유지 겹침**(Figma 50% 투명도 위에): 최종 판정용. 8배 확대로 본다.
- **색 제거 위치 겹침**(파랑=우리/빨강=Figma/보라=일치): 색 차이·렌더링 차이에 속지 않고 순수 위치만 볼 때.

### 5. 보고

리포트(HTML artifact): 영역별로 [① 우리 화면 | ② Figma | 겹침(투명도 슬라이더)] 3열, 사진 영역 회색 마스킹, 라벨은 색 태그로 명확히("① 우리 화면(지금 코드)" / "② 피그마 원본"). 티켓별 수정 내역 표 + 테스트 결과.

## 안티패턴 (이 세션에서 실제로 저지른 실수들)

- ❌ 수치만 보고 "일치"라고 보고 (→ 육안 겹침이 판정 기준)
- ❌ 리포트 뷰포트(1180px)에서 캡쳐해 900px 이미지로 겹침 (→ 크기 불일치)
- ❌ 가로만 측정하고 세로 오프셋을 놓침
- ❌ 색 판정식이 다른 두 요소를 비교해 가짜 1px을 "보정" → 멀쩡한 밑선 파괴
- ❌ 눈대중으로 gap을 늘렸다 줄였다 (→ 반드시 잉크 실측 후 계산된 값만 적용)
- ❌ 0.5px position 이동 시도 (→ 크롬 스냅으로 무효, 실측으로 확인됨)
- ❌ 경계 AA 픽셀(±1 flutter)을 맞추려 반복 수정
- ❌ Figma 구버전 색을 "정답"으로 적용 (→ 색은 사용자/디자이너 확인 필수)
